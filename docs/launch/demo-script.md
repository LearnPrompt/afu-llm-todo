# 60 秒 Demo 脚本

## 目标

用 Sample Vault 演示 `收件箱 -> Wiki -> 行动卡 -> 题库 -> 周排期`，让第一次看到项目的人明白阿福不是普通 Todo，而是 Obsidian 前面的内容管家。

## 准备

```bash
npm run demo:reset
npm run demo
open http://localhost:4317
```

确认首屏显示 Sample Vault。

## 镜头 1：问题

画面：工作台停在 `收件箱`。

旁白：

> Obsidian 很适合积累，但收件箱很容易变成“以后再看”。阿福要做的不是再建一个 Todo，而是把素材变成可执行安排。

操作：

- 展示 3 条 sample 收件箱素材。
- 勾选 2-3 条。
- 点击 `加入 Wiki 批次`。

## 镜头 2：Wiki

画面：切到 `Wiki`。

旁白：

> Wiki Mode 会先把一批素材整理成可回链的 Markdown Wiki，再从里面生成行动卡。

操作：

- 点击 `编译收件箱批次`。
- 点击 `生成行动卡`。
- 展示行动卡里的原因、来源和置信度。

## 镜头 3：接受行动卡

画面：Wiki 行动卡列表。

旁白：

> 这里不是让 LLM 直接替你排满日程，而是先给出可接受、可拒绝的行动建议。

操作：

- 接受 1 张行动卡。
- 点击 toast 的 `查看题库` 或手动切到 `题库`。

## 镜头 4：题库

画面：题库出现刚接受的卡。

旁白：

> 接受后的行动卡会变成 Obsidian 选题卡，Markdown 仍然是唯一真源。

操作：

- 展示卡片标题、摘要、标签和来源。
- 点击 `排期` 或拖拽到周历。

## 镜头 5：周排期

画面：排期弹窗。

旁白：

> 最后选一个时间段，写回 Markdown，也可以同步到飞书或 macOS 日历。

操作：

- 选择 `下午制作 14:00-15:30`。
- 保存排期。
- 展示周历中出现卡片。

## 结尾

旁白：

> 这就是 Afu：Inbox to Wiki, Wiki to Todo, Todo to Calendar.

画面：

- 回到 README 或首屏 logo。
